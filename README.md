# نظام بطاقات العضوية - الجمعية الكويتية للمختبرات الطبية (KAML)

## 🎯 **نظرة عامة**

نظام شامل لإدارة بطاقات العضوية مع ربط Firebase لحفظ البيانات في السحابة. النظام يعمل كصفحة ثابتة على GitHub Pages مع قاعدة بيانات حقيقية.

## 🌐 **الموقع المباشر**

```
https://kaml-system.github.io/KAML-CARD/
```

## 📋 **الصفحات المتاحة**

### 🏠 **الصفحة الرئيسية**
- `https://kaml-system.github.io/KAML-CARD/`
- عرض معلومات النظام والخدمات المتاحة

### 📱 **ماسح الباركود**
- `https://kaml-system.github.io/KAML-CARD/scanner.html`
- متاح للجميع بدون تسجيل دخول
- مسح باركود البطاقات والتحقق من صحتها
- عرض بيانات العضو من قاعدة البيانات

### 🔐 **تسجيل دخول المسؤولين**
- `https://kaml-system.github.io/KAML-CARD/login.html`
- تسجيل دخول للمسؤولين المصرح لهم
- نظام مصادقة Firebase

### 🛠️ **إدارة البطاقات**
- `https://kaml-system.github.io/KAML-CARD/admin.html`
- متاح فقط للمسؤولين بعد تسجيل الدخول
- إنشاء بطاقات عضوية جديدة
- رفع صور الأعضاء
- حفظ البطاقات في Firebase Storage

## 🔥 **إعدادات Firebase**

### **الخدمات المستخدمة:**
- ✅ **Firestore Database** - قاعدة البيانات
- ✅ **Authentication** - نظام المصادقة
- ✅ **Storage** - تخزين الصور والبطاقات
- ✅ **Analytics** - تحليلات الاستخدام

### **Collections في Firestore:**
- `members` - بيانات الأعضاء
- `admins` - بيانات المسؤولين
- `cards` - معلومات البطاقات المُنشأة

## 🛡️ **نظام الأمان**

### **🔐 نظام المصادقة:**
- تسجيل دخول المسؤولين محمي بـ Firebase Auth
- التحقق من صلاحيات المسؤول قبل الوصول للصفحات المحمية
- إعادة توجيه تلقائية للمستخدمين غير المصرح لهم

### **🛠️ حماية البيانات:**
- جميع البيانات محفوظة في Firebase
- قواعد الأمان في Firestore لحماية البيانات
- تشفير البيانات الحساسة

## 🚀 **التثبيت والنشر**

### **1️⃣ إنشاء مستودع GitHub:**
```bash
اسم المستودع: kaml-card
النوع: Public
```

### **2️⃣ رفع الملفات:**
- رفع جميع الملفات إلى المجلد الرئيسي
- التأكد من وجود `index.html` في الجذر

### **3️⃣ تفعيل GitHub Pages:**
- Settings → Pages
- Source: Deploy from a branch
- Branch: main
- Folder: / (root)
- Save

### **4️⃣ إعداد Firebase:**
- إنشاء مشروع Firebase جديد
- تفعيل Firestore Database
- تفعيل Authentication
- تفعيل Storage
- نسخ إعدادات Firebase إلى `firebase-config.js`
